Att arbeta med Sass

Den största och mest uppenbara fördelen med Sass över vanlig css är att man kan få till en mycket bättre struktur. Med bara css är allting på samma nivå och det gör att det är lätt att tappa bort sig, även om det inte är fråga om en särskilt stor mängd rader. Med Sass kan man både använda sig av partials för att separera till exempel olika undersidor eller typer av element. Man kan även använda sig av nesting så att man enkelt kan se vad som hör ihop eller inte.

En annan fördel är att man kan skriva mindre repetitiv kod eftersom man kan använda variabler, inheritance och så vidare vilket både gör koden kortare och underlättar när man ska göra ändringar på flera ställen samtidigt. Jag ser inga direkta nackdelar med att använda Sass idag. Det är kompatibelt med det mesta och inte svårt att sätta sig in i om man redan kan css (eller för den delen om man inte kan det).

Att börja jobba med Sass efter att bara skrivit vanlig css-kod tidigare var lite av en omställning. Man får se upp så att man inte råkar falla in i att skriva som man är van, men även om man råkar falla in i gamla vanor så är det lätt att komma tillbaka och helt enkelt peta in vanliga css-en i Sass-strukturen. Man kan också implementera den förändringen stegvis vilket underlättar. Jag tror att med lite övning så kommer mycket av ordningsstrukturen att kännas väldigt naturlig.
